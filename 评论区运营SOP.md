# 论文降AI类评论区运营SOP

> 基于"论文降AI热评库"1288条真实评论数据，提炼可执行的评论区运营标准流程。

---

## 一、运营目标

在小红书平台，围绕"论文降AI/AIGC检测"主题的笔记评论区，通过**评论布局、热评筛选、精准回复**三大手段，引导用户认知、建立信任、驱动转化。

**核心指标：**
- 热评命中率（是否符合热评三大标准）
- 评论互动率（点赞数、子评论数）
- 评论引导转化（从评论到私信/咨询的行为）

---

## 二、运营全流程（五步法）

### 第一步：评论监控与采集

**触发条件：** 发布新笔记后，或发现竞品/相关话题高热度笔记后

**操作清单：**

| 动作 | 工具/方式 | 频率 |
|------|----------|------|
| 监控目标笔记评论区 | 小红书APP + 飞书多维表格手动录入 | 笔记发布后24h内高频（每2-4h），之后每日1次 |
| 采集评论数据并录入飞书表 | 截图/复制粘贴 → 飞书表格 | 实时 |
| 记录笔记基础信息 | 原笔记链接、笔记文本内容、发布时间 | 每篇笔记1次 |

**采集字段标准：**
- `评论内容`：完整复制评论文本
- `评论人`：用户名 + 是否标注 [作者] 标签
- `评论类型`：主评论 / 子评论
- `被评论人`：（子评论时填写）
- `点赞数`：当前点赞数
- `子评论数`：当前回复数量
- `地区`：评论者IP属地
- `原笔记链接`：完整URL

**质量要求：**
- 每条评论必须采集完整文字内容（含emoji）
- 子评论需标注被回复人
- 评论人用户名后标注 `[作者]` 标记博主回复身份

---

### 第二步：评论分类与打标

**在飞书多维表格中完成以下分类：**

#### 2.1 评论内容类型打标（三选一）

| 类型 | 定义 | 示例 |
|------|------|------|
| **吸引点击类** | 引起共鸣、制造好奇、情绪表达、幽默吐槽 | "滕王阁序ai率60%，王勃找谁说理去" |
| **询问类** | 提出具体问题、求助、咨询 | "paperpass免费版降到15%以下，知网就是20%以下了吗？" |
| **效果反馈类** | 分享使用体验、工具推荐、方法教学 | "用AI，输入指令直接降就好了啊，根本用不着出钱" |

**操作标准：**
- 吸引点击类 = 数据中占比最高（64.8%），是主航道内容
- 询问类 = 高转化潜力，需要重点关注回复
- 效果反馈类 = 热评率最高（29.6%），需要重点培育

#### 2.2 评论顺序角色打标（五选一）

| 角色 | 功能定位 | 说明 |
|------|---------|------|
| **首评提问** | 引导讨论方向、抛出锚点问题 | 笔记发布后第一条评论，设置讨论基调 |
| **第二三条产品讨论** | 植入产品/方法讨论 | 紧随首评，制造"群众讨论"氛围 |
| **第四五条痛点/复购讨论** | 唤醒痛点、刺激复购冲动 | 深化话题，激发真实用户参与 |
| **普通评论** | 填充评论区活跃度 | 自然的互动和讨论 |
| **非引导评论** | 标记需处理的负面/广告评论 | 建议删除或举报 |

#### 2.3 回复处理状态打标（四选一）

| 状态 | 触发条件 | 操作 |
|------|---------|------|
| **必须回复** | 首评提问中的高价值问题；询问类且提及竞品比较；引发争议的评论 | 24h内以作者身份回复 |
| **选择性回复** | 效果反馈类中的正面评价；点赞数增长快的普通评论 | 72h内选择性以作者身份回复 |
| **建议删除** | 广告引流评论；恶意攻击；竞品恶意刷评 | 立即删除 |
| **无需回复** | 普通互动评论；纯情绪表达无需跟进 | 不操作 |

**回复处理状态 × 角色交叉策略（基于数据验证）：**

| 角色 | 必须回复 | 选择性回复 | 建议删除 | 无需回复 |
|------|---------|-----------|---------|---------|
| 首评提问 | 33% | 67% | 0% | 0% |
| 第二三条产品讨论 | 4% | 14% | 0% | 82% |
| 第四五条痛点复购 | 1% | 19% | 0% | 80% |
| 普通评论 | 2% | 23% | 0% | 75% |
| 非引导评论 | 0% | 0% | 100% | 0% |

### 第三步：热评识别与标记

#### 3.1 热评三大判定标准（满足任一即标记为热评）

| 标准 | 定义 | 判断方式 |
|------|------|---------|
| **点赞数全局前10%** | 该评论点赞数 > 全部1288条评论点赞数的90分位值 | 飞书表格中排序比较 |
| **热度分全局前10%** | 该评论热度分 > 全部评论热度分的90分位值 | 飞书表格中排序比较 |
| **笔记内热度排名前3** | 该评论在所属笔记中的热度排名 ≤ 3 | 按笔记分组排序 |

**热评数据特征（基于1288条样本）：**
- 热度分中位数：7.55 → 热评门槛约 ≥ 22.00（Q3值）
- 点赞数中位数：1 → 热评门槛约 ≥ 5（Q3值）
- 热评率：14.5%（187/1288）
- 热评中效果反馈类占比最高（29.6%热评率）

#### 3.2 热评每周刷新流程

1. **数据刷新：** 每周一更新所有评论的点赞数、子评论数
2. **热度计算：** 按飞书表格公式自动计算热度分
3. **排名更新：** 按笔记分组，更新笔记内热度排名
4. **阈值更新：** 重新计算全局点赞数前10%和热度分前10%阈值
5. **热评标记：** 满足任一条件则标记 `是否热评 = true`，记录 `热评原因`

#### 3.3 热评原因分类维护

飞书表格字段 `热评原因`（多选），支持组合标记：
- `点赞数进入全局前10%`
- `热度分进入全局前10%`
- `笔记内热度排名前3`

### 第四步：评论布局与执行

#### 4.1 笔记发布后评论布局SOP（72小时黄金窗口）

**时间线：**

```
笔记发布后0-2h：
  → 首评提问上线（由运营小号发布）
     - 内容类型：询问类
     - 目标：设置讨论锚点，引导后续评论方向
     - 示例："用的什么查重系统？知网还是维普？"

笔记发布后2-8h：
  → 第二三条产品讨论上线
     - 内容类型：吸引点击类 + 效果反馈类
     - 目标：制造"真实讨论"氛围
     - 示例："我部分AI写的30%AI率，人工改半天一查升到50%多，笑了"

笔记发布后8-24h：
  → 第四五条痛点/复购讨论上线
     - 内容类型：效果反馈类 + 吸引点击类
     - 目标：深化痛点、插入解决方案
     - 示例："用AI输入指令直接降就好了，根本不用出钱"

笔记发布后24-72h：
  → 普通评论持续填充
  → 选择性以作者身份回复高价值评论
  → 删除非引导评论中的广告/恶意评论
```

#### 4.2 评论角色话术模板

**首评提问模板：**
- "知网查的AI率多少算安全？15%以下可以吗？"
- "paperpass免费版降AI靠谱吗？和知网差距大吗？"
- "有没有人用维普查AI率的，准不准？"

**第二三条产品讨论模板：**
- "我部分AI写的30%AI率，然后人工吭哧吭哧改半天，一查升至50%几"（吸引点击类，数据已验证有效）
- "抄的有多爽，降的就有多绝望……"（原文标题植入评论区）
- "用deepseek改，先把标红部分一段段改，再上传知网复查"（效果反馈类）

**第四五条痛点/复购讨论模板：**
- "常用的降重方法有删除、扩写、缩写、打乱语序、多语言翻译等，如果改不动可以试下XXX"（效果反馈类）
- "我觉得最有效的，还是得一段一段改，理论红了就让AI写一遍，翻译没用"
- "请将这段话改写，通过调整语序增减字数，替换同义词，缩写再扩写"

#### 4.3 回复执行标准

**以作者身份回复时：**
1. 必须回复：24h内完成，使用真诚语气，不机械套话
2. 选择性回复：72h内完成，侧重转化引导
3. 回复内容中可自然提及品牌/产品名，避免硬广感
4. 回复时点"赞"同评论，提升评论热度

**禁止操作：**
- 使用完全相同的回复模板（会被检测）
- 在回复中直接放链接或联系方式
- 批量短时间内刷评

## 六、附录

### A. 热度分计算公式（建议）

```
热度分 = 点赞数 × 0.6 + 子评论数 × 0.3 + 发布时间衰减系数 × 0.1
```
（具体公式需根据飞书表格实际公式确认）

### B. 非引导评论处理SOP

满足以下条件标记为"非引导评论"并建议删除：
1. 包含明确外部链接或联系方式
2. 与主题完全无关的广告内容
3. 恶意攻击或人身攻击
4. 竞品恶意刷评（相同内容高频出现）

### C. 评论人标记说明

- `[作者]` = 博主/官方账号
- 不带标记 = 普通用户
- 多个 `[作者][作者]...` 可能因数据导入格式导致，实际表示以作者身份回复
